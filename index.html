HTML

  • HTML stands for Hypertext Markup Language.
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is the type of content on a webpage.
  • Some common elements used to add content to pages are text elements, image elements, header elements, and list elements.
  • Some common elements used to divide or describe a page are meta-related elements, content divsion elements, and sematic elements.
  • The head element contains information about the webpage, it is not visible to the user.
  • Metadata is contained in the head element, and it is data about our webpage that the browser uses to effectively serve our webpage to users.
  • Inside the head element, you can use the metadata tags such as title, style, base, meta, script.
  • The head element sets our character encoding, an HTTP reader for content, and the default view for our webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about an element. It is always defined in the beginning element and never in the closing element.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git.
  • git push: transmit local branch commits to the remote repository branch.
  • git pull: fetch and merge any commits from the tracking remote branch.
  • git status: checks what branch we are currently on.
  • git checkout: switch to another branch and check it out into your working directory.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git clone [url]: retrieve an entire repository from a hosted location via URL.
