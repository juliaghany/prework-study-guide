HTML

  • HTML stands for Hypertext Markup Language.
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is the type of content on a webpage.
  • Some common elements used to add content to pages are text elements, image elements, header elements, and list elements.
  • Some common elements used to divide or describe a page are meta-related elements, content divsion elements, and sematic elements.
  • The head element contains information about the webpage, it is not visible to the user.
  • Metadata is contained in the head element, and it is data about our webpage that the browser uses to effectively serve our webpage to users.
  • Inside the head element, you can use the metadata tags such as title, style, base, meta, script.
  • The head element sets our character encoding, an HTTP reader for content, and the default view for our webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about an element. It is always defined in the beginning element and never in the closing element.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git.
  • git push: transmit local branch commits to the remote repository branch.
  • git pull: fetch and merge any commits from the tracking remote branch.
  • git status: checks what branch we are currently on.
  • git checkout: switch to another branch and check it out into your working directory.

CSS

  • CSS: "Cascading Style Sheet" is a programming language used by developers to define how webpage content is presented to users. It defines the colors, aesthetics, visual layout, and organization.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS: used when a developer wants to style an element directly in the HTML file. This is useful for quick and permanent change, but less flexible than internal/external style sheets because each inline style must be edited individually when you make a design change.
  • Internal CSS style sheet: when you embed an entire stylesheet directly in the HTML file using a style element in the head element. This contains CSS rules for the webpage in the head section of the HTML file. A single change to the CSS rule will apply to all tagged page elements.
  • External CSS style sheet: this is the most commmon way to style a webpage. It allows developers to keep all their CSS rules in a seperate file, which makes design changes easier. When using this, we link the file using a link element placed in the HTML's head.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Using a * symbol applies a rule to all the elements visible on the page.
  • When you combine two selectors into a single CSS rule with a comma, you allow the declarations to affect both selectors. This avoids unnecessary lines of code and repetition, which is also known as DRY (Do Not Repeat Yourself) code.
  • Text-align is a positioning property and is one way to center a title or heading on a webpage.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The link element creates the connection to the external file.
  • Selector: defines the element or attributes to which the rules, or declarations, will apply.
